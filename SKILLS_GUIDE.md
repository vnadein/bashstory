# Skills для разработки BAJOUR Social

## Обзор

Skills - это специализированные руководства, которые помогают в разработке конкретных частей проекта. Они содержат шаблоны, лучшие практики и справочную информацию.

## Расположение

Skills находятся в директории `~/.qwen/skills/`:

```
~/.qwen/skills/
├── bashstory-commands/
│   └── SKILL.md      # Разработка команд терминала
├── bashstory-db/
│   └── SKILL.md      # Работа с базой данных
├── bashstory-ui/
│   └── SKILL.md      # UI компоненты
└── bashstory-testing/
    └── SKILL.md      # Тестирование и отладка
```

## Когда использовать каждый skill

### bashstory-commands
**Использовать когда:**
- Создаёте новую команду для терминала
- Модифицируете существующую команду
- Добавляете алиасы команд
- Нужна справка по API команд

**Примеры запросов:**
- "Создай команду `stats` для показа статистики пользователя"
- "Добавь алиас `cls` для команды `clear`"
- "Как реализовать команду с вводом пароля?"

---

### bashstory-db
**Использовать когда:**
- Нужен SQL запрос для получения данных
- Отладка проблем с базой данных
- Просмотр схемы БД
- Оптимизация запросов

**Примеры запросов:**
- "Покажи все посты пользователя с комментариями"
- "Как получить ленту постов для пользователя?"
- "Очисти все сессии в БД"

---

### bashstory-ui
**Использовать когда:**
- Изменяешь терминальный UI
- Добавляешь новые визуальные эффекты
- Работаешь с стилями Tailwind
- Нужен шаблон React компонента

**Примеры запросов:**
- "Добавь анимацию мигания для курсора"
- "Создай компонент для отображения постов"
- "Как изменить цвет терминала?"

---

### bashstory-testing
**Использовать когда:**
- Тестируешь новую функциональность
- Отладка проблем
- Нужны тестовые сценарии
- Проверка производительности

**Примеры запросов:**
- "Как протестировать команду `follow`?"
- "Почему не сохраняется сессия?"
- "Запусти production сборку"

---

## Интеграция с MCP-серверами

Skills работают в связке с MCP-серверами для максимальной эффективности:

```
┌─────────────────────────────────────────────────────────┐
│                    Ваш запрос                           │
└─────────────────────────────────────────────────────────┘
                            │
                            ▼
┌─────────────────────────────────────────────────────────┐
│  Выбранный Skill (контекст и шаблоны)                   │
│  - bashstory-commands                                   │
│  - bashstory-db                                         │
│  - bashstory-ui                                         │
│  - bashstory-testing                                    │
└─────────────────────────────────────────────────────────┘
                            │
                            ▼
┌─────────────────────────────────────────────────────────┐
│  MCP Серверы (выполнение действий)                      │
│  - filesystem: чтение/запись файлов                     │
│  - sqlite: SQL запросы                                  │
│  - git: коммиты, ветки                                  │
│  - terminal: запуск команд                              │
│  - puppeteer: браузерные тесты                          │
└─────────────────────────────────────────────────────────┘
```

## Примеры использования

### Создание новой команды

**Запрос:** "Создай команду `weather` для показа погоды"

**Процесс:**
1. Skill `bashstory-commands` предоставляет:
   - Шаблон команды
   - Список файлов для изменения
   - Примеры реализаций

2. MCP `filesystem`:
   - Читает существующие команды
   - Создаёт новый файл `lib/commands/implementations/weather.ts`

3. MCP `terminal`:
   - Запускает `pnpm dev` для тестирования

---

### Отладка БД

**Запрос:** "Покажи последние 5 постов с комментариями"

**Процесс:**
1. Skill `bashstory-db` предоставляет:
   - SQL шаблоны запросов
   - Структуру таблиц

2. MCP `sqlite`:
   - Выполняет SQL запрос
   - Возвращает результаты

---

### UI изменения

**Запрос:** "Добавь поддержку Markdown в вывод команд"

**Процесс:**
1. Skill `bashstory-ui` предоставляет:
   - Информацию о компоненте TerminalOutput
   - CSS классы для стилизации
   - Примеры использования marked

2. MCP `filesystem`:
   - Читает компонент
   - Вносит изменения

3. MCP `puppeteer`:
   - Делает скриншот для проверки

---

## Расширение skills

### Добавление нового skill

1. Создать директорию:
```bash
mkdir -p ~/.qwen/skills/bashstory-newfeature
```

2. Создать SKILL.md с документацией

3. Добавить ссылки в существующие skills при необходимости

### Темы для новых skills

Возможные расширения:
- **bashstory-api** - REST API endpoints
- **bashstory-auth** - Аутентификация и авторизация
- **bashstory-deploy** - Деплой и CI/CD
- **bashstory-performance** - Оптимизация производительности

---

## Best Practices

### 1. Используйте правильный skill
Выбирайте skill по типу задачи:
- Команды → bashstory-commands
- Данные → bashstory-db
- UI → bashstory-ui
- Тесты → bashstory-testing

### 2. Комбинируйте с MCP
Для сложных задач используйте несколько MCP серверов:
```
filesystem + sqlite = миграция данных
terminal + git = commit после тестов
```

### 3. Обновляйте skills
При добавлении новой функциональности:
- Обновите соответствующий skill
- Добавьте примеры использования
- Документируйте новые файлы

### 4. Следуйте шаблонам
Skills содержат проверенные шаблоны:
- Используйте их как основу
- Адаптируйте под задачу
- Сохраняйте стиль кода

## Справочник

### Ключевые файлы проекта

```
lib/commands/
├── dispatcher.ts         # Маршрутизация команд
├── types.ts              # Типы команд
└── implementations/      # Реализации команд

lib/
├── db.ts                 # Подключение к БД
├── social.ts             # Социальные функции
└── session.ts            # Сессии

components/terminal/
├── terminal.tsx          # Главный компонент
├── components/           # Подкомпоненты
└── constants.ts          # Константы

app/api/command/
└── route.ts              # API endpoint
```

### Команды для разработки

```bash
# Разработка
pnpm dev                 # Запуск dev-сервера
pnpm build               # Production сборка
pnpm start               # Production сервер

# Качество кода
pnpm lint                # ESLint проверка
npx tsc --noEmit         # TypeScript проверка

# БД
sqlite3 data/quotes.db   # Прямой доступ к БД
```
